# Marvel Tour — обобщение на проекта

> Контекстен документ за Claude Project „марвел". Описва какво представлява сайтът, как е устроен и как се поддържа. Кодът живее локално и в GitHub — този файл е само резюме за контекст.

---

## 1. Какво е сайтът

Marvel Tour е сайт на туристическа агенция от Пловдив — клонинг/преработка на marveltourbg.com. Показва оферти за почивки, екскурзии и екзотика, дестинации, блог, контакти, и интеграции на живо с външни платформи.

- **Тип:** чист статичен сайт — само **HTML + CSS + JavaScript**, без билд стъпка, без framework, без сървър.
- **Без Node/Python** в средата (python е Store-stub). Скрейпинг и помощни скриптове са писани на **Perl**.
- **Език на интерфейса:** български.

## 2. Хостинг и деплой

- **GitHub repo:** `https://github.com/rachamatheus/marvel.git`
- **Хостинг:** GitHub Pages, който сервира от клона **`site`**.
- Работен клон: **`main`**.
- **Деплой команда:**
  ```bash
  git push origin main          # запазва кода
  git push origin main:site     # публикува на живо (GitHub Pages)
  ```
- След деплой сайтът се обновява за ~1 минута. Кеширане се бие с `?v=NN` версии (виж по-долу).

## 4. Cache-busting версии (актуални)

Всеки asset се зарежда с `?v=NN`. При промяна на файл се вдига номерът, за да не сервира браузърът стара версия. Текущи:

```
styles.css?v=151
app.js?v=157
oferta.js?v=154
lang.js?v=136
data/offers.js?v=137
data/hotels/<num>.js?v=2
favicon.svg?v=9
logo.svg?v=7
admin.js?v=134
```

**Правило:** редактираш файл → вдигаш неговото `?v=` навсякъде, където е реферирано → commit → push към `main` и `main:site`.

## 5. Ключови интеграции и решения

### Изображения през proxy
Снимките минават през `wsrv.nl` за надеждност/скорост (заобикаля hotlink защити):
`https://wsrv.nl/?url=<encoded>&w=1400&output=webp&we&q=82`. При неуспех — retry директен URL, после placeholder.

### PeakView / Evertour (оферти на живо)
- `jivo.html` вгражда външни iframe-и с `iframeResizer`:
  - `garcia` → Хотели в Гърция (`iframe.peakview.bg/garcia/?cl=999`)
  - `turcia` → Хотели в Турция
  - `yacht` → Яхтен туризъм (`evertour.art.bg`)
- За клиентите лентата „Powered by PeakView" е скрита с бяла ивица (`.jivo-cover`); в **админ таба остава видима**.
- Cross-origin съдържанието не може да се променя — затова покритие с CSS, а не редакция.
- Самият marveltour също е на PeakView (партньор) → не може да се копира интеграцията им; нашият път е white-label iframe.

### Карта на дестинациите
- Equirectangular карта на света, `viewBox "0 0 1010 510"`.
- 63 дестинации с точки (`DEST_COORDS`); име на дестинацията се показва **само при hover**.
- Континентите са върнати към оригиналните размери (без зуум/изрязване).

### Footer „Информация" страници
- 6-те линка (общи условия, политика, застраховки, трансфери, ваучер, индивидуални екскурзии) **отварят наши страници** (`info.html?p=...`), не препращат към marveltour.
- Текстът е свален и изчистен от marveltour в `data/pages/*.html`.

### Оферти — брой
- Публичният сайт прави dedup: персонализирани оферти (`mt_custom_offers` в localStorage) **заместват** базовите по id; зачитат се изтритите id-та. Така общият брой = базови + персонализирани, без двойно броене.

### Транспорт етикети
`plane/flight → ✈️ Полет`, `bus → 🚌 Бус`, `car → 🚗 Кола`.

### Любими
След добавяне в любими, презареждането връща в началото на страницата (не към офертата).

## 6. Постоянни правила за работа (важни)

- **Навигационната лента трябва да е еднаква на всяка страница** — промяна в навбара се прилага навсякъде.
- **Dropdown менютата трябва да са винаги напълно видими** (без отрязване): `minmax(0,1fr)` + `min-width:0` + ellipsis, да остават в екрана; запазват цвета си при скрол.
- **Не се въвеждат пароли/логин данни** в чужди панели (отказано е по политика и по желание на собственика).
- Флагчета — реални изображения (`flagcdn.com`), не emoji (на Windows emoji флагове чупят).

## 7. Среда за локален преглед

Няма Python. За локален сървър се ползва малък Perl скрипт `_serve.pl` (виж `.claude/launch.json`), стартиран през preview инструмента на порт 8099.
